Advanced Algorithm (ETHz)

Highlights of the course advanced algorithm.

Approximation Algorithm

A algorithm with approximation rate $\alpha$ to solve an optimization problem: the output of the algorithm is at most $\alpha\cdot OPT$ where $OPT$ is the minimization goal (or at least $OPT/\alpha$ when $OPT$ is the maximization goal).

Set Cover

Given $m$ sets with weights over an $n$ elements universe, find the minimum weights sets that cover the universe. This problem is known to be NPC.

Use $\Delta$ to denote the maximum number of elements contained in a single set. Use $f$ to denote the maximum number of sets that covers a single element. We have the following approximation algorithms.

Set Cover
There exists a polynomial Set Cover algorithm with approximation rate $H_{\Delta}=O(\log \Delta)$.
There exists a polynomail Set Cover algorithm with approximation rate $f$, when weights are fixed to $1$.
Proof.

First Algorithm: Suppose there are $k$ elements which are not covered. Pick a set that cover $l$ of them with weight $c$ where $c/l$ is smallest, repeate this procedure. Since $OPT$ costs are enough to cover all $k$ elements, there must exists a set with $c/l\le OPT/k$. Treat picking a set as putting $c/l$ weight in each of the newly added elements. Thus, the total cost is at most $OPT\cdot H_n$.

A better analysis: Suppose $S$ is a set with weight $c$ in OPT and suppose at some point $k$ elements in $S$ are not covered, where $k\le\Delta$. Since picking $S$ lead to $c/k$ weights putting in each newly added elements, the next step to cover some elements in $S$ must put weights at most $c/k$ in each newly added elements. Thus, the sum of weights putting on each elements in $S$ is at most $c\cdot H_\Delta$. By summing up all sets in $OPT$, the sum of weights on every elements is at most $OPT\cdot H_ {\Delta}$.

$O(OPT\cdot \log\Delta)$ is also the worst case for this algorithm. Consider $\log\Delta$ sets where each contains $2^i$ elements where $i\in[\log\Delta]$, and another three sets each contains roughly $\frac{1}{3}$ of every sets. All the sets has weight $1$. The above algorithm will pick $\log\Delta$ sets, since each time we can shrink the uncovered elements to half size, while pickin each of the last 3 sets will only shrink it to $1/3$. However, the best solution is $3$.

Second Algorithm: For each element that is not covered by any picked sets, pick all the sets that contain the elements, call this element as critical element. Repeat this procedure. One can see that critical elements have disjoint neibors, while OPT must contains one set among neibors of each critical elements. Thus, the number of sets is $OPT\cdot f$.

This approximation is optimal.

[DS14, Corollary 1.5]
For any $\epsilon>0$, approximate Set Cover with rate $(1-\epsilon)\cdot \ln n$ where $n$ is the number of elements, is NP-hard.

Knapsack

Given $n$ items, each with a value and cost. Given $t$, find a subset of itmes that has the larges sum of values, while the sum of costs are at most $t$.

One can solve this probelm trivially through dynamic programming, by letting $M[i,j]$ as the smallest cost of itmes in the first $i$ items, that has sum of value $j$. Denote the maximum value as $p$, then the running time is $O(n^2p)$.

However, this problem is NP-hard (since by letting values equal to weight this problem becomes subset sum). Nonetheless, there is a good approximation algorithm.

Knapsack
There is an FPTAS (for any $\epsilon$ the algorithm take $\epsilon$ as input and approximate with rate $1+\epsilon$ in time polynomial on the size and $\frac{1}{\epsilon}$) for Knapsack.

By rounding down all values to the nearest multiple of a interger $k$, we get a loss of at most $nk$, which lead to approximation rate $1+(nk)/p$, since OPT is at least $p$. To get an $(1+\epsilon)$ approximation, $k$ should be setting to $p\epsilon/n$, and the running time becomes $O(n^3/\epsilon)$.
