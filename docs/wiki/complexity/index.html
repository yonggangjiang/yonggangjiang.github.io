Basics

Turing Machine and Computation

One possible definition of a $k$-tape Turing machine $M=(Q,\Sigma,\Gamma,\delta,q_0,Q_{acc})$
1. $Q$ is a finite set, the states.
2. $\Sigma$ is a finite set, the input alphabet.
3. $\Gamma$ is a finite set, the tape alphabet. $\Box\in\Gamma$ is the blank symbol, $\Sigma\subseteq\Gamma\backslash\{\Box\}$.
4. The transition function $\delta: Q\times\Gamma^k\to Q\times \Gamma^k\times\{L,S,R\}^k$.
5. $q_0\subseteq Q$ is the start state.
6. $Q_{acc}\subseteq Q$ is the set of accepting states.
The first tape is the input tape. For non-determinisitic, the transition function will map to multiple states.

A configuration $C$ is $(q,(p_1,x_1),…,(p_k,x_k))$, denoting the state and positions, all information on tapes. $p_i$ are positions and $x_i$ is a string. $x_i$ only contain positions where the machine has reached. New positions will be added to $x_i$ if the pointer arrives at that position.

We use $C\vdash_M C’$ to denote $C$ can get $C’$ by applying one step of $\delta$. $SC_M(x)$ is the start configuration on input $w$, then $SC_M(x)\vdash_M C_1\vdash_M C_2 … \vdash_M C_t$ is the computation of $M$ on $w$, denoted as $SC_M(x)\vdash_M^*C_t$.

Time Complexity

The computation is unique on $SC_M(x)$. Denote the length of the computation as $Time_M(x)$. For an integer $n$, define

\[Time_M(n):=\max\{Time_M(x)\mid |x|=n\}\]

Let $t:\mathbb{N}\to\mathbb{N}$ be a function. Then $M$ is $t$ time bounded means $Time_M(n)\le t(n)$ for all $n$.

Space complexity

The space of a configuration is $Space(C)=\max_{1\le\kappa\le k}|x_k|$.

Similar to time complexity, we can define $Space_M(x)=\max{Space(C)}$ where $C$ are all configurations on the computation. We can also define $Space_M(n)$ for integer $n$. Note that even if $M$ does not halt, the space can possibly be infinite.

Time classes

A language (or called $0-1$ function) $L$ is deterministically t time decidable, if there exists a Turing machine $M$ such that $L(x)=M(x)$ for any $x$ and $M$ is $t$-time bounded.

$DTime(t)$ contains all $L$ such that $L$ is deterministically t time decidable. (There is a definition $DTime_k(t)$ where the turing machine is restricted to $k$ tapes.)

$DSpace$ is similarly defined.

Simple Lower Bound.

Let the language $LEN={a^nb^n\mid n\in\mathbb{N}}$/ $LEN$ is not in $DSpace(o(\log n))$.

Let $COPY={w cw\mid w\in{a,b}^*}$. Then $COPY\not\in DTime_1(o(n^2))$.

An intersting theorem
$DSpace(o(\log \log n))=DSpace(O(1))$.

Complexity Classes

Reduction

The following is an abstract definition of many-one reduction.

Many-one reduction
Let $R$ be a set of functions $R: \Sigma^ *\to\Sigma^*$. $L'$ is $R$ many-one reducible to $L$ if
\[\exists f\in R,\forall x\in\Sigma^*, x\in L'\leftrightarrow f(x)\in L\]
$L$ is $C$-hard with respect to $R$ many-one reduction if for any $L'\in C$, $L'$ is $R$ many-one reducible to $L$. If further $L\in C$, then $L$ is $C$ complete with respcet to $R$ many-one reduction.

To choose a good reduction class $R$ and a complexity class $C$, two conditions are reasonable: 1. if $L’$ can be reduced to $L\in C$, then $L’\in C$ 2. transitivity of reduction.

Another definition besides many-one reduction.

Turing reduction
For a function $f:\Sigma^*\to\Sigma^*$, an oracle turing machine $M^f$ has a seperate oracle tape, an additional query state $q_?$ and answer state $q_!$. When $M^f$ enter $q_?$, then content of the oracle tape is replaced by $f(x)$ from $x$ and $M$ enters $q_!$. $L'$ is Turing reducible to $L$ if there is a oracle turing machine $R^L$ such that $L'(x)=R^L(x)$ for any $x$.

Turing reduction is not closed under NP

Co-classes

For a problem class $C$, co-C is the set containing all $\bar{L}$ where $L\in C$. Here $\bar{L}(x)=1-L(x)$ for any $x$.

theorem
For any ? space constructible function $s(n)\ge \log n$, we have $NSpace(s)=co-NSpace(s)$.

L and NL

We use logspace many-one reductions, where $R$ is defined to be all the functions that can be computed by logspace turing machine: a read-only input tape, logspace bounded work tapes and oneway write-only output tape. It is transitive and if $L\le_{log}L’\in C$, then $L\in C$ for $C=L,NL,P$. So it is a good reduction definition for $L,NL,P$.

Directed reachability (CONN) is NL-complete. So $\bar{CONN}$ is co-NL-complete. By proving $\bar{CONN}\in NL$, it gives the tollowing theorem. See Immerman–Szelepcsényi theorem.

theorem
$NL=co-NL$.

Circuits

Definitions

Boolean function maps $n$-bits to $m$-bits. Boolean circuit is an acyclic digraph computes a boolean function. Boolean formula forces all nodes to have outdegree at most $1$. To covert a boolean circuit of depth $d$ to an equavalent boolean formula, the upper bound and lower bound are both $2^{\Theta(d)}$.

Families of circuitls is a sequence of circuits $\mathcal{C}=C_1,C_2,…$ such that $C_i$ has $i$ inputs. For functions $s,d$, $\mathcal{C}$ is $s$ size bounded and $d$ depth bounded if $size(C_i)\le s（i)$ and $depth(C_i)\le d(i)$ for all $i$. $\mathcal{C}$ computes a function $f:{0,1}^*\to{0,1}$ if $C_{|x|}(x)=f(x)$ for any $x$.

$\mathcal{C}$ is called $s$ space and $t$ time constructible, if there is an $s$ space bounded and $t$ time bounded DTM that on input $1^n$ constructs an encoding of $C_n$.

A $s$ size bounded $\mathcal{C}$ is called logarithmic space uniform if it is $O(\log s)$ space constructivble. It is called polynomial time uniform, if it is $poly(s)$ time constructible.

$log-unif-DepthSize(d,s)$ contains all language $L$ that can be determined by a $d$ depth and $s$ size bounded logarithmic space uniform family of circuits. $P-unif-DepthSize(d,s)$ is defined similarly.

Relation to Turing machine

Circuits to DTM algorithm
If $L\subseteq \{0,1\}^*$ is decided by a $d$ depth bounded and $s$ space constructible family of circuits C, then $L \in DSpace(d + s)$. If it is $s$ size bounded and $t$ time constructible, then $L \in DTime(t + s\log^2s)$.
DTM algorithm to circuits
Let $t$ be ? time constructible and let $L \subseteq \{0,1\}$ be in $DTime(t)$. Then there is a $O(t^2)$ time constructible family of circuits that is $O(t^2)$ size bounded and decides $L$. The family is also $O(log t)$ space constructible for reasonable function $t$. Let $L\subseteq \{0,1\}^*$ be in $NSpace(s)$. Then there is a $s$ space constructible family of circuits that is $O(s^2)$ depth bounded and decides L.

The above theorem is due to the following observation about oblivious turing machine (the heads move in the same way on all inputs with length $n$).

Lemma about oblivious DTM
For every $t$ time bounded DTM $M$ there is an oblivious $O(t^2)$ time bounded 1-tape DTM $S$ with $L(M) = L(S)$.

So the input lenth determines the circuit (which simulates the inputs and output for each head reads).

We have $NSpace(s)$ has a circuit that is $s^2$ depth bounded and $s$ space constructible, which gives a NTM in $s^2$ space. This gives the following Savitch’s theorem.

Savitch’s theorem
$NSpace(s)\subseteq Space(s^2)$.

Based on this theorem, we get Nick’s class $NC_k:=\cup_{i\in\mathbb{N}}log-unif-DepthSize(\log^k(n),O(n^i))$ and $NC:=\cup_{i\ge 1}NC_i$ and the relationship

\[NC_1\subseteq L\subseteq NL\subseteq NC_2\subseteq NC_3....\subseteq NC\subseteq P\]

P-complete problem

Recall that logspace many-one reduction is closed downward under $L,NL,P$. Actually, it is also closed in any $NC_i$ and $NC$. So to research the paralizability of $P$, we define $P$-complete under the logspace many-one reduction. Now whether or not $P$-complete problem is in $NC$ determines whether or not $P$ is equal to $NC$.

$P$-complete problem is believed to have no good paralized algorithm (in $NC$). Such as, CVAL (compute the value of a circuit on an input), maximum flow, ,depth-search, breadth-depth search (which stores explored nodes in a min-heap, each time take a minimum node to add all its neibors to the heap, ask whether a node is visited before another node).

Whether or not Perfect matching is $P$-complete or in $NC$ is an open problem.

For more $P$-complete problems, see Limits to Parallel Computation.

Polynomial Method

$AC$ is similar to $NC$, where $AC$ has unlimited fan-in, but $NC$ has constant fan-in. On can see that $NC_i\subseteq AC_i\subseteq NC_{i+1}$. $NC_0$ is not interesting since it only computes constant inputs. $AC_0$ is the lowest that is interesting to prove lower bound. Our goal in this section is the following lower bound.

Parity lower bound
Parity is not in $AC_0$, i.e., there are no polynomial size and constant depth unbounded fan-in circuit that computes parity. More precisely, any constant depth $d$ circuit that computes parity on $n$ inputs has size at least $\Omega(\sqrt[2d]{n})$ . ny polynomial size circuit that computes parity on $n$ inputs has depth at least $\Omega(\log n/ \log \log n)$.

The proof use polynomial method (boolean function analysis). First prove no polynomial with degree less than $\sqrt{n}$ can approximate parity funcion, then prove constant depth circuit can be approximate by a polynomial with small degree.

Branching Program

$BP_k$ contains all width $k$ branching program. A branching program is a decition tree: a layered directed graph, where the first layer has source node $s$, the last layer has $accept$ and $reject$. Each node corresponds to an input bit, and has two out edge, one is $0$ and one is $1$.

Barrington Theorem
For any $k$, we have $BP_k\subseteq NC_1$. We also have $NC_1\subseteq BP_5$.

Polynomial time hierarchy

Define $\Sigma^P_k$ as the language where $x$ is in the language is equavelent to $\exists y_1\forall y_2….: R(x,y_1,y_2,…,y_k)=1$, here $R$ is decided by a polynomial turing machine. Similarly, $\Pi^P_k$ is $\forall y_1\exists y_2….: R(x,y_1,y_2,…,y_k)=1$. So $NP=\Sigma^P_1$ and $co-NP=\Pi^P_1$.

We define $PH=\cup_{k\ge 1}\Sigma^P_k$, and $PH\subseteq PSPACE$.

If $\Sigma^P_i=\Pi^P_i$ for some $i$, then $PH=\Sigma^P_i=\Pi^P_i$ for any $i$.

The complete problem for $\Sigma^P_k$: given a boolean formula $F$, whether or not $Q_1x_{i_1},…F(x_1,..,x_n)$ is true, where $Q$ alternates for $k$ times.

$\Sigma^P_i$ is equevalent to $NP$ given any problem in $\Sigma^P_{i-1}$ or $\Pi^P_{i-1}$ as an oracle, $\Pi_i^P$ is co-NP given $\Sigma^P_{i-1}$ or $\Pi^P_{i-1}$ as an oracle.

Turing machines with advice

Problems that can be computed by a polynomial size circuit family is equavalent to the following class of problems. Both are denoted as $P/poly$.

there exists a polynomial time turing machine $M$ and a function $\alpha:\mathbb{N}\to{0,1}^{poly}$, such that $M$ determine the problem with $\alpha(|x|)$ written on the tape.

Karp–Lipton theorem
If $NP\subseteq P/poly$, then $\prod^P_2\subseteq \sum^P_2$.

P, NP and PSPACE

Under polynomial many-one reduction, True quantified Boolean formula is PSPACE-complete.

Probablitstic computation

computation classes

RTime(t) contains all problems $L$ that can be determined by a $t$ time randomized turing machine such that: for $x\in L$, $M$ accept with probability $\ge\frac{1}{2}$ and for $x\not\in L$, $M$ rejects $L$ with probability $1$.

BPTime(t) has two sided error, where both has probability $\ge\frac{2}{3}$.

Define $RP=\cup_{i\in\mathbb{N}}RTime(n^i), BPP\cup_{i\in\mathbb{N}}BPTime(n^i), ZPP=RP\cap co-RP$. Notice that ZPP is the algorithm that has zero error but with randomized running time.

Relationship: $P\subseteq ZPP\subseteq RP/co-RP\subseteq BPP\subseteq PSPACE\subseteq EXP$.

Relationship to NP is subtle: we know $RP\subseteq NP$ because a lauguage is accepted if and only if there exists a random string to make the probability turing machine to accept the language (if it is not in the language, then machine will never receive it.) However, the relationship of BPP and NP is vague.

Karp–Lipton theorem
$BPP\subseteq P/poly$

Probabilitic amplification

We can write $BPP$ as $BP-P$. In general $BP$ is an operation. The definition is: for a class $C$, $BP-C$ contains language $L$ that exists a $B\in C$ such that $x\in L$ if the probability that $x$ applies a random string is in $B$ is at least $\alpha+\beta/2$; the other case is $\alpha-\beta/2$.

Positive Turing Reduction remember that a turing reduction is an oracle turing machine $M^A$ that determines $B$ (in which case $B$ is reduced to $A$). Moreover, $M^A$ is called positive if for any $A\subseteq A’$, we have $L(M^A)\subseteq L(M^{A’})$, that is, string accepted by $A$ must be accepted by $A’$. The advantage: NP is closed under positive turing reduction, but not turing reduction.

Moreover, if $C$ is closed under positive turing reduction, then $BP-C$ allows probability amplification.

Finially, we have $BPP\subseteq \prod^P_2\cap\sum^P_2$, see Sipser–Lautemann theorem .
