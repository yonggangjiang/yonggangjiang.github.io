D1 Software Admin Guidance

Ste-by-step guidance on the responsibilities of a D1 software admin.

Create Accounts for Newcomers

Upon receiving the account-creating request e-mail from our secretary (usually the e-mail is sent to both the admin and the newcomer). There are three important pieces of information: the newcomer’s name, the newcomer’s current e-mail, and the newcomer’s rule (Ph.D. student? Vister? Postdoc?).

Step 1: Create an account request.

Go to https://domino.mpi-inf.mpg.de/ and click “Request a new account”. Then check the correct person, click “Proceed”, and a form will appear. There is only one place we need to change, which is “Email (Alias) Group(s)”, check “d1-phd” or “d1-postdocs” or “d1-students” or “d1-guest” according to the newcomer’s role (interns are in the category “d1-students”). Then click “Request a new account” and “Complete the request”.

Remark: “Unix login” is also a place we can change (it is the login username) if the newcomer has his/her own preference.

Step 2: send information to the newcomer.

After finishing the request, we need to wait for several hours until we receive an e-mail with the title “[Account: Receipt for Requester]…”

Download this script, change the global variable ID, password, name as your e-mail login username, password and your first name. In the same folder of this script run command 

python send_email.py {newcommer’s first name} {newcommer’s e-mail}

You can find the newcomer’s e-mail from the e-mail you received from our secretory (for example, a Gmail).

Update D1 Activity

Sometimes our secretary might send an e-mail to inform you to make changes to our group website “activity”.

Login to https://www.mpi-inf.mpg.de/typo3/. Go to Page -> “IST TYPO3” -> “Algorithms & Complexity” -> “Activity”. Do the changes in the corresponding box (usually travel).

I have talked with Danupon and currently we don’t need to update the “noon semiar” as it is already in our event calander.

Account expiration

You will constantly receive e-mails titled “MPI-INF account expiration warning…”. If someone replied this e-mail and said he/she wants to extend the account, go to the intranet, click “IDM group manager”, and

Change e-mail group

This is also done in the “IDM group manager”, currently there are two situations: 1. someone wants to be added to the noon-seminar group 2. construct a conspiracy group for the hat making.
